# MCP 授權規範 · 第 2 部分  
# 授權伺服器發現（Authorization Server Discovery）

## 1. 為什麼需要「發現」？

MCP 用戶端第一次連線 MCP 伺服器時：

- 它不知道授權伺服器在哪裡  
- 不知道 Token 要去哪裡換  
- 不知道支援哪些 scope  
- 不知道是否支援 PKCE  
- 不知道是否需要動態註冊  

因此 MCP 採用 **OAuth 2.0 授權伺服器中繼資料（RFC 8414）**  
讓用戶端能自動找到授權伺服器。
